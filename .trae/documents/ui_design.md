# UI/UX 设计文档

## 1. 设计风格

* **核心理念**: 极简、现代、内容优先。

* **色彩系统**:

  * 主色调: Zinc (中性灰)，搭配少量强调色 (如 Indigo 或 Violet)。
    -不仅支持明暗模式 (Light/Dark Mode)，且需平滑切换。

* **字体**: 使用系统字体栈或 Inter 字体，保证阅读体验。

## 2. 页面规划

### 2.1 访客端 (Public)

* **首页 (Home)**

  * 顶部导航栏 (Logo, 首页, 归档, 关于, 搜索, 主题切换)。

  * Hero 区域 (简介或最新推荐)。

  * 文章列表 (卡片式布局，显示标题、摘要、日期、标签)。

  * 底部页脚 (版权信息, 友链)。

* **文章详情页 (Post Detail)**

  * 文章元信息 (作者, 时间, 阅读量)。

  * Markdown 内容渲染 (排版优美，代码高亮)。

  * 目录导航 (TOC)。

  * 评论区 (输入框 + 评论列表)。

* **归档页 (Archives)**

  * 按时间轴或分类/标签展示文章索引。

* **关于页 (About)**

  * 博主介绍，联系方式。

### 2.2 管理端 (Admin)

* **仪表盘 (Dashboard)**

  * 访问统计概览。

  * 最新评论待审核提醒。

* **文章编辑器 (Editor)**

  * 全屏/宽屏布局。

  * 左侧: Markdown 编辑区域。

  * 右侧: 实时预览区域 (Sync Scroll)。

  * 顶部工具栏: 发布、存草稿、设置封面/标签。

* **内容管理 (Content Management)**

  * 文章列表 (表格视图，支持搜索和批量操作)。

  * 评论管理 (审核通过/拒绝)。

## 3. 交互与动画 (Framer Motion)

* **路由切换**: 页面间切换时使用淡入淡出 (Fade) 或滑入 (Slide) 效果。

* **列表加载**: 使用 Stagger 动画逐个显示文章卡片。

* **骨架屏 (Skeleton)**: 数据加载时显示占位骨架，避免布局跳动。

* **微交互**: 按钮点击缩放、链接悬停下划线动画。

## 4. 组件库 (shadcn/ui)

* Button, Input, Select, DropdownMenu, Dialog, Toast, Card, Avatar, Badge, Skeleton.
